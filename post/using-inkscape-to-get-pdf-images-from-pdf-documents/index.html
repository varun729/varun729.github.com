Using Inkscape to get PDF images from PDF documents

I use PDF images in my LateX documents. Creating PDF images from Python and gnuplot is trivial because of the rich API features. However, I sometimes generate plots in Microsoft Excel, where images can only be saved as PNG, JPG or other pixel formats and not in any of the vector formats. (Excel on Mac OS X allows saving images in PDF format)

##Workaround I found a workaround for getting PDF images from Microsoft products. Following are the steps that I use. I am only explaining the steps here in as much detail as text allows.

Step 1. Export the document in PDF format. The image will be exported as a vector object, but will be part of the parent document. In order to get the image in PDF to use in LateX, we need to do more. If we select the image in Microsoft Excel, and then export the document, then the PDF will only contain the image, but the image will be smaller than the document size.

Step 2. Open the PDF document in Inkscape. There is an option for \“Precision of approximating gradient meshes\“, which we set to the maximum or very fine.

Step 3. Press Ctrl+A to select the image in the document. If there are other contents in the document, delete them by selecting them individually.

Step 4. After selecting the image, click on \“File > Document Properties\“. In \“Custom Size\“, expand \“Resize page to content…\“. Click on the button \“Resize page to drawing or selection\“.

Step 5. Save the document in Inkscape (we opened PDF, so the PDF will be overwritten). You can also \“Save as\” another file.

These steps will allow cropping PDF documents to get a PDF image within the document. This is much better than cutting the image after opening the PDF in a PDF document viewer such as Adobe Acrobat viewer.
