Atmega8 features RISC architecture. From its name we can interpret that the number of instructions available to use it will be very limited, but will allow to create any complex algorithm using a combination.

Below is a summary from what I understood by reading : The RISC Concept - A Survey of Implementations

RISC(Reduced Instruction Set Computer)

  1. The instruction set must be kept simple
  2. The instructions should run in a minimal time
  3. Pipelining is more important than the size of the code
  4. Compilers should be optimized to transfer complexity from the hardware to the compiler phase

Simply, RISC traits are known as:

  • small instruction set
  • load/store architecture
  • fixed length coding and hardware decoding
  • large register set
  • delayed branching
  • processor throughput of once instruction per cycle at an average

Calling a processor RISCy just on the basis of the instruction set size, is giving incomplete information. A processor should satisfy the 'one instruction per cycle on average' requirement.

Phases in a execution path of an instruction:

  • instruction fetch
  • decode
  • operation fetch
  • ALU execution
  • memory access
  • write back the results

A sequential processor, can keep the execution path such that different parts of the execution path for different instructions can be executed simultaneously. This is parallelism at a single processor level. This is a core feature of RISC processors, and is achieved by designing a suitable instruction set for pipelining.

The different stages of a pipeline needs to be independent of each other so that they can be executed in parallel. Also, each stage should take the same time, referred to as one clock cycle. The deeper the pipeline (more the number of stages per instruction), easier is to achieve parallelism, but it complicates the overall design.

If different instructions require different size of bits to be read, the pipeline stages will become dependent upon the size of the instruction (time required to fetch). This can be overcome by having a buffer, which pre-fetches, but easier approach would be to have all instructions designed such that they are of the same size. Similar decoding scheme for the instructions also will make it more simpler to implement (MIPS is a RISC architecture which uses this). Having to read register/memory takes some extra time, which if can be avoided will help in reducing the pipeline stalling.

Load/Store architecture is a feature of the RISC. 'load' means loading the contents from memory into a register, and 'store' is the reverse. They take one clock cycle to execute, but if in the design, they are used a couple of clock cycles later, they will be surely available to be used. In case they are still not available pipeline will stall till available. The compiler usually handles the scheduling of when the register should be used.
